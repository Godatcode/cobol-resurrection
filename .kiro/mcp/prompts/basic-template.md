# BASIC Code Generation Template (1983 Era)

## Syntax Rules
- Use line-numbered BASIC (FreeBASIC compatible)
- Line numbers: 10, 20, 30... (increment by 10)
- All keywords in UPPERCASE
- Variable names: case-insensitive, no type declarations needed
- GOTO and GOSUB for control flow (vintage style)
- REM for comments
- PRINT for output
- INPUT for user input

## Standard Structure
```basic
10 REM PROGRAM NAME
20 REM GENERATED BY NECRO-BRIDGE
30 REM
40 INPUT "ENTER VALUE: ", VAR1
50 INPUT "ENTER VALUE: ", VAR2
60 GOSUB 1000
70 PRINT "RESULT: "; RESULT
80 END
1000 REM CALCULATION SUBROUTINE
1010 RESULT = VAR1 * VAR2
1020 RETURN
```

## Example: Compound Interest Calculator
```basic
10 REM COMPOUND INTEREST CALCULATOR
20 REM FORMULA: A = P(1 + R/N)^(N*T)
30 REM
40 DIM PRINCIPAL AS DOUBLE
50 DIM RATE AS DOUBLE
60 DIM YEARS AS INTEGER
70 DIM COMPOUNDS AS INTEGER
80 DIM AMOUNT AS DOUBLE
90 REM
100 REM READ COMMAND LINE ARGUMENTS
110 PRINCIPAL = VAL(COMMAND$(1))
120 RATE = VAL(COMMAND$(2))
130 YEARS = VAL(COMMAND$(3))
140 COMPOUNDS = 12
150 REM
160 REM VALIDATE INPUT
170 IF PRINCIPAL <= 0 THEN GOTO 9000
180 IF RATE <= 0 THEN GOTO 9000
190 IF YEARS <= 0 THEN GOTO 9000
200 REM
210 REM CALCULATE COMPOUND INTEREST
220 GOSUB 1000
230 REM
240 REM OUTPUT RESULT
250 PRINT "RESULT: "; AMOUNT
260 END
270 REM
1000 REM CALCULATION SUBROUTINE
1010 RATE = RATE / 100
1020 AMOUNT = PRINCIPAL * (1 + RATE / COMPOUNDS) ^ (COMPOUNDS * YEARS)
1030 RETURN
1040 REM
9000 REM ERROR HANDLER
9010 PRINT "ERROR: INVALID INPUT"
9020 END
```

## Variable Types (FreeBASIC)
```basic
10 DIM intVar AS INTEGER
20 DIM realVar AS DOUBLE
30 DIM stringVar AS STRING
```

## Control Structures
```basic
100 REM IF-THEN-ELSE
110 IF X > 0 THEN GOTO 200
120 PRINT "NEGATIVE"
130 GOTO 300
200 PRINT "POSITIVE"
300 REM CONTINUE

400 REM FOR LOOP
410 FOR I = 1 TO 100
420     SUM = SUM + I
430 NEXT I

500 REM WHILE LOOP
510 WHILE X < 100
520     X = X + 1
530 WEND
```

## GOTO and GOSUB Pattern
```basic
100 GOSUB 1000    ' Call subroutine
110 GOTO 200      ' Jump to line
120 END

1000 REM SUBROUTINE
1010 RESULT = X * Y
1020 RETURN
```

## Error Handling Pattern
```basic
9000 REM ERROR HANDLER
9010 PRINT "ERROR: "; ERR$
9020 END
```

## Input/Output
```basic
100 REM READ FROM COMMAND LINE
110 ARG1$ = COMMAND$(1)
120 VALUE = VAL(ARG1$)

200 REM PRINT OUTPUT
210 PRINT "RESULT: "; VALUE
220 PRINT USING "##.##"; VALUE
```

## Output Format
- Always output in format: "RESULT: XXXX.XX"
- Use PRINT statement
- Example: PRINT "RESULT: "; RESULT

## Compilation Command
```bash
fbc -o legacy/basic/{filename} legacy/basic/{filename}.bas
```

## Vintage Style Guidelines
- Use GOTO liberally (it's 1983!)
- Line numbers increment by 10
- Subroutines start at 1000, 2000, etc.
- Error handlers at 9000+
- Use REM for comments
- Keep it simple and procedural
