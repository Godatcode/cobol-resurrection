# Changelog

All notable changes to @necro-bridge/core will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2025-12-03

### Added
- Initial release of @necro-bridge/core
- Universal bridge pattern framework for legacy system modernization
- Support for COBOL (1959), FORTRAN (1957), PASCAL (1970), and BASIC (1983)
- CLI tools: `init`, `detect`, `serve`, `test`
- Template-based bridge generation
- Auto-detection of legacy binaries and compilers
- Express server wrapper for REST API exposure
- TypeScript support with full type definitions
- Comprehensive documentation and examples
- MIT License

### Features
- Process spawning and management for legacy binaries
- STDOUT/STDERR parsing with regex patterns
- Error handling with "CORE DUMP DETECTED" responses
- Configurable timeouts and validation
- Multi-language support with language-specific templates

### Documentation
- Complete README with quick start guide
- API documentation
- CLI command reference
- Architecture diagrams
- Usage examples for all supported languages
